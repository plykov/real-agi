# FastBank — an AI capability-transfer program for an enterprise, packaged as a repeatable consulting pipeline

> **Up:** [Verticals — the portfolio](../README.md) · **Russian counterpart:** [`Ru/3_Verticals/fastbank/README.md`](../../../Ru/3_Verticals/fastbank/README.md)

> **Part of the Real AGI portfolio.** A domain probe in organizational AI transformation: it touches the navigator concept as a *mentoring model executed by a human consultant*, not yet by an agent over the core. **Stage:** drafts — a real client engagement captured as documents; the conveyor lives in the author's practice, not in code.

FastBank is a content project that packages an AI-consulting offer for a large enterprise client — "from diagnosis and needs articulation to an implementation program and a contractual frame." Inside: a meeting summary with the client's leadership, a department-by-department diagnostic questionnaire, a multi-week program to train internal "champions," an execution program, a priced offer, and a draft service agreement. The same frame has already been re-applied to a second client, which is the evidence that the method is portable.

The job it does: take an organization from "we use a fraction of these tools and our staff fear AI" to a signed program that builds internal capability. Who it's for: the client's managers and champions (who must learn to apply AI operationally), and the consultant (whose goal is to close the deal).

## How it realizes the concept

The program *is* a mentoring model: organizational goal → obstacles (security/compliance, duplicated initiatives, primitive tool use) → competence deficits → guided progression to results via cases, office hours, and train-the-trainer. That is the navigator's shape — reality model, scenario space, correction from labeled consequences — but the one leading the client is a human "playing coach," not the system. On the ladder it sits at *pipeline*: a repeatable chain (meeting → diagnosis → program → offer → contract), with one coded step and a second designed. It is not yet an agent: no automated feedback loop, no cross-client self-learning — reuse across clients is manual transfer.

## What exists today

Built: the full document set, a real negotiation with the client's leadership, fixed pricing tiers, a draft agreement, a PDF-build script, and a second-client adaptation. Designed but not built: the diagnostic survey web app (spec only — no frontend/backend in the repo). The pipeline's steps are still produced ad-hoc in chat sessions.

## Where it could go

*Hypothesis:* if the leading role moves from consultant to an agent built on the Real AGI core — the diagnostic questionnaire becoming the domain knowledge base, the program becoming scenario space — this becomes a genuine vertical, "AI transformation of organizations," that diagnoses, sequences, and coaches at scale. Today that core is not used; the value proven here is that the *method* generalizes across clients.
