# Ontology — a fictional universe as a knowledge base, and the proto-language of the core

> **Up:** [Verticals — the portfolio](../README.md) · **Russian counterpart:** [`Ru/3_Verticals/ontology/README.md`](../../../Ru/3_Verticals/ontology/README.md)

> **Part of the Real AGI portfolio.** Not a vertical and not an engine: this is a domain knowledge base — a worldbuilding corpus plus an AGI-book draft — that quietly rehearses the core's reality model and supplies ontological language for the rest of the corpus. **Stage:** drafts (text only, no code).

Ontology is a philosophical-ontological knowledge base and lore-builder for an invented universe: a hierarchy of levels of being, programmable "reality protocols," an ethics code, institutions, and a recursion wall — alongside draft chapters of a book about AGI. The product is text: lore, essays, and chapters, developed through dialogue with AI models. Its founding thesis is deliberately extreme — that physical reality does not exist, and all reality is consciousness / information / code.

## How it realizes the concept

This project sits at the *domain* layer, not the engine layer. It builds no navigator, no memory, no scenario fan, no self-learning loop — so on the agent ladder it is below "agent." What makes it part of the one concept is structural resonance: the lore contains proto-figures of the core. A "growth program" and a "big quest" (guiding a candidate toward full status) mirror *a mentor leading a client to a goal*; "smart plots" mirror *the scenario space*; the overseer figures mirror a *positioning/supervision layer*. It is, in effect, a hand-built ontology of "what a reality and a guided trajectory look like" — the kind of formal world-picture the core would later need to represent any domain.

## What exists today

A layered text corpus (numbered folders): base ontology, fundamental concepts, the invented civilization and realm, institutions, and a large question-source document acting as the request queue. There is a documented but manual editorial pipeline: open question → test the thesis against several AI models → curate → integrate into concept files → book chapters. Feedback exists (multiple models critique the same claims); automation, self-learning, and a defined market do not.

## Where it could go

*Speculation:* this corpus could become the domain knowledge base for a content vertical (a book or course on AGI and simulation ontology) running on top of the shared core, and a donor of precise ontological vocabulary to the Real AGI corpus itself. To climb the ladder it would need a real end consumer with a goal, a guiding mechanic instead of static text, and the manual "question → model run → synthesis → integration" loop scripted into an actual pipeline. All of that is hypothesis, not current state.
