# Questions — движок структурированного интейка, который строит модель реальности из людей

> **Вверх:** [Вертикали — портфель](../README.md) · **English counterpart:** [`Eng/3_Verticals/questions/README.md`](../../../Eng/3_Verticals/questions/README.md)

> **Часть портфеля Real AGI.** Это шаг *интейка / диагностики*, который питает модель реальности навигатора, — диалоговый опросник, превращающий то, что люди знают, делают и чего боятся, в структурированные данные. Он пока никого не ведёт; он собирает фактическую основу, которую ведение бы потребовало. **Стадия:** рабочий продукт (развёрнут, протестирован), единственный клиент.

Questions — это приложение-опросник, построенное для оценки готовности организации к внедрению AI. Анонимные респонденты проходят двухэтапные опросники в диалоговом формате «один вопрос за раз» — этап диагностики и этап исполнения — с автосохранением, ретраями на нестабильной сети и восстановлением сессии. Ответы консолидируются в базу данных, которую аналитик использует, чтобы оценить масштаб и спроектировать программу обучения и предложение.

Он был построен из конкретной потребности: диагностировать AI-зрелость корпоративного клиента по подразделениям, ролям, текущему внедрению и ограничениям комплаенса — вместо того чтобы угадывать по интервью и общим формам.

## Как это реализует концепцию

Ядро навигатора начинается с *модели реальности*: точной картины того, где пользователь на самом деле находится. Questions — это ровно тот первый шаг, реализованный для организаций, а не для индивидов, — систематический механизм выявления текущего состояния, дефицитов и ограничений. На лестнице он стоит низко и делает это честно: чистый **пайплайн** (начать сессию → отвечать с автосохранением → завершить → консолидировать → экспортировать), ещё не агент. Нет ни веера сценариев, ни петли обратной связи между этапами, ни самообучения, ни фигуры, которая ведёт. Он фиксирует входные данные; он ещё не решает и не рекомендует.

## Что есть сегодня

Продакшен-система: одностраничный фронтенд, serverless-бэкенд, версионируемая конфигурация опросника, анонимные сессии на токенах, автосохранение с ретраями, админ-дашборд с экспортом в CSV и unit/integration/e2e-тесты. Построено и развёрнуто для одного клиента.

## Куда это может вырасти

*Гипотеза:* переиспользуемая часть — это диагностический движок, а не специфичный для клиента контент. Он мог бы стать интейк-фронтендом вертикали «корпоративной AI-трансформации» — где результаты первого этапа формируют второй (петля обратной связи), собранная диагностика питает автоматизированный шаг анализа-и-рекомендации, а опыт накапливается между клиентами. Эта траектория — путь от сборщика к навигатору; сегодня существует только сборщик.
