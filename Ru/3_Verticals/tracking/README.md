# Tracking — навигатор, повёрнутый на самого себя: ранжирование того, куда вложить самый дефицитный ресурс — время

> **Вверх:** [Вертикали — портфель](../README.md) · **English counterpart:** [`Eng/3_Verticals/tracking/README.md`](../../../Eng/3_Verticals/tracking/README.md)

> **Часть портфеля Real AGI.** Концепция, направленная на собственный портфель проектов автора, — стратегический навигатор, ранжирующий ставки по тому, куда должно идти время. Сам этот портфельный анализ есть ручная версия того, что tracking автоматизирует. **Стадия:** работает (реальные вызовы API, постоянное хранилище, применяется на собственном портфеле автора); персональный инструмент, пока не рыночный продукт.

Основателю, у которого больше идей, чем времени, нужен навигатор для самого высокоставочного решения из всех — *в какую траекторию вложить самый дефицитный ресурс, время*. Инструмент начинался как трекер проектов и лаунчер утилит, но в коде вырос в систему поддержки стратегических решений: он берёт каждый проект в рабочем пространстве, превращает сырые идеи в предпроектные досье, прогоняет по ним веб-исследование рынка, оценивает каждый AI-аналитиком по девяти стратегическим критериям и собирает ранжированный лидерборд с вердиктом убить/продолжить и дедлайном первого сигнала. Фигура ментора здесь ведёт автора через его собственный портфель — не «что это за проект», а «куда тебе на самом деле стоит идти дальше и почему».

## Как это реализует концепцию

Здесь используются все три органа ядра, применённые к самому пространству проектов. *Модель реальности:* каждый проект как элемент с состоянием. *Пространство сценариев:* каждая сырая идея развёрнута в досье с рыночным исследованием. *Позиционирование:* AI-аналитик оценивает каждый по взвешенным стратегическим критериям, а детерминированный шаг кода сворачивает оценки в композитный ранг фокуса с вето-кэпом на риск зависимости от вендора — суждение модели, учёт коду. Пайплайн полный (идеи → досье → исследование → оценка → ранжирование), и ведущая функция явно прописана в промптах («назначь горизонт, дедлайн первого сигнала, правило убить/продолжить»). Чего ему не хватает — замкнутой петли последствий: исходы ставок ещё не размечаются и не возвращаются обратно, чтобы перекалибровать веса критериев — они пока заданы вручную.

## Что есть сегодня

Рабочий инструмент с реальными вызовами модели (с веб-поиском), постоянным хранилищем и живым UI, применяемый на собственном портфеле автора. README по-прежнему описывает только более старый слой трекера-и-лаунчера; стратегический движок живёт в коде, построен позже. Чего недостаёт для полной вертикали: рынка за пределами автора (это персональный стратегический инструмент) и самообучения на серии решений.

## Куда это может вырасти

*Гипотеза:* это самодемонстрация архитектуры. Корпус говорит, что навигатор помогает человеку держать продуктивный вектор по измерениям жизни; tracking — это тот самый навигатор, направленный на измерение «какая из моих ставок заслуживает моего времени». Добавьте бизнес-обвязку и петлю последствий — и работающий персональный навигатор станет вертикалью: стратегическим вторым пилотом для любого соло-основателя, выбирающего, куда ставить.
