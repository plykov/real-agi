# essays-claude — самообучающаяся агентная организация: цикл ядра «учиться на последствиях», в действии

> **Вверх:** [Вертикали — портфель](../README.md) · **English counterpart:** [`Eng/3_Verticals/essays-claude/README.md`](../../../Eng/3_Verticals/essays-claude/README.md)

> **Часть портфеля Real AGI.** Слой ядра-движка: место, где цикл «решение → результат → коррекция» не просто спроектирован, а *работает* на реальных размеченных прогонах. Это самая конкретная в портфеле реализация самообучающегося органа ядра. **Стадия:** прототип (40+ записанных прогонов в двух рантаймах; продуктовый контур спроектирован, но ещё не доведён до живых пользователей).

essays-claude — экспериментальная площадка, где строится автономная мультиагентная «организация»: наблюдатель (агент в роли CEO) проводит две независимые рантайм-песочницы через каноническую серию бенчмарк-задач и накапливает уроки в общем ядре. Его цель — production-ready агентная организация, которая берёт реальную задачу и доводит её до результата без человека. Работают два многошаговых цикла: экспериментальный (золотая задача → бриф запуска → автономный прогон → статус/скоркард → перенос уроков в ядро) и спроектированный продуктовый (запрос на проект → нормализация намерения → продуктовый бриф → инженерная спецификация → граф задач → исполнение).

## Как это реализует концепцию

Почти весь проект *и есть* универсальная машинерия — в этом его содержание: пайплайн-движок (плоскость исполнения, оркестратор, непрерывный цикл), слой памяти/состояния, сценарии и контрольные задачи (золотые задачи), обратная связь (скоркарды, находки) и самообучение (методология плюс реальные следы обучения). Домен, к которому он привязан, — автономная разработка ПО — намеренно остаётся лишь тестовой ареной; механизмы (вести через директивы, оценивать последствия решений, корректировать) переносятся на любую вертикаль. Цикл «решение → результат → коррекция» присутствует и конкретен: директива запускает прогон, скоркард его оценивает, уроки правят ядро и контракты, следующая волна проверяет исправление. Задокументированные доменные препятствия — честные препятствия автономности: рантаймы застревают между прогонами, бесконечные циклы переотправки, зависание без прогресса, отсутствие честного терминального отказа, остановка на первом же трении вместо обхода.

## Что есть сегодня

Рабочий прототип: десятки реальных прогонов в двух рантаймах, пройденный набор бенчмарков и инфраструктура автономности (непрерывный цикл, heartbeat, протокол наблюдатель↔рантайм, бюджеты ретраев, честные состояния отказа). Чего нет: продуктовый контур, который провёл бы человека-неразработчика от намерения к доставленному приложению, спроектирован, но ещё не запущен на живых пользователях; а бизнес-слой (кто платит, цена, выход на рынок) отсутствует намеренно — это эксперимент с ядром, а не вертикаль.

## Куда это может вырасти

*Гипотеза:* essays-claude — один из трёх разрозненных кусков ядра — самообучение, — доказанный на 40+ реальных прогонах. Сплавленный с долговременной памятью ([MaaS](../maas/README.md)) и антипараличным сценарным циклом ([autonomy-hub](../autonomy-hub/README.md)) в единый движок, он поставляет ту часть, которую корпус называет самой трудной: систему, которая оценивает последствия собственных решений и меняет саму себя. Доведённый до живых пользователей, его продуктовый контур заодно породит и собственную вертикаль класса A — автономную dev-организацию для неразработчиков.
