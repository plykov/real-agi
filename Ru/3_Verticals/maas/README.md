# MaaS — Memory as a Service: долговременная память ядра, построенная как самостоятельная инфраструктура

> **Вверх:** [Вертикали — портфель](../README.md) · **English counterpart:** [`Eng/3_Verticals/maas/README.md`](../../../Eng/3_Verticals/maas/README.md)

> **Часть портфеля Real AGI.** Слой ядра-движка: долговременная семантическая память, которая нужна навигатору, построенная как переиспользуемый сервис, а не как часть какой-то одной вертикали. Два из трёх членов формулы *агент = пайплайн + обратная связь + самообучение* — память и самообучение — живут здесь. **Стадия:** прототип (MVP работает сквозным образом; подсистема самообучения по большей части спроектирована, но ещё не построена).

MaaS — это событийно-управляемая система долговременной памяти для LLM: она запоминает контекст прошлых диалогов и вплетает его в новые ответы, решая проблему «каждый разговор начинается с чистого листа». Запрос проходит через явный конечный автомат — проанализировать, нужна ли память, и извлечь семантические ключи, собрать релевантные воспоминания в контекст, сгенерировать ответ, записать диалог в лог — а фоновый архивариус периодически сжимает сырые логи в размеченные, реферированные записи памяти. Её прямой потребитель — код (REST API для разработчиков, строящих AI-ассистентов); человек появляется лишь как конечная точка чата.

## Как это реализует концепцию

Это почти целиком универсальная машинерия — именно поэтому она ядро, а не вертикаль: у неё нет домена (память работает над любым диалогом) и она не ведёт человека к цели. Вклад в ядро здесь фундаментальный: четырёхслойная память траектории, которую описывает первое эссе корпуса, событийно-управляемый движок пайплайна с горячей заменой промптов и спроектированный контур самообучения (сенсор → аналитик → учитель → тюнер) с метриками, золотыми датасетами и экспериментами. Концептуальные материалы прямо позиционируют MaaS как подсистему памяти «AI-эксперта/ментора» — спроектированный фрагмент архитектуры супер-ментора. На лестнице рантайм — это рабочий одиночный пайплайн; петля обратной связи/самообучения специфицирована, но её агенты — пока пустые папки.

## Что есть сегодня

Рабочий MVP: сквозной пайплайн запроса, эмулятор пользователя и анализатор на стеке Node/TypeScript + Postgres + LLM. Описана трёхуровневая бизнес-модель (SaaS-память, лицензия для self-hosted, расширенный пакет самообучения). Чего не построено: роли самообучения (аналитик, учитель, тюнер, менеджер) существуют как документация и пустой каркас; нет данных о живых пользователях и выручке; а известное слабое место — модель не всегда использует извлечённые воспоминания — отмечено, но не устранено.

## Куда это может вырасти

*Гипотеза:* MaaS — один из трёх разрозненных кусков ядра — память, — которые портфель находит уже построенными, но ещё ни разу не собранными вместе. Сплавленный с петлёй сценариев/анти-паралича ([autonomy-hub](../autonomy-hub/README.md)) и петлёй самообучения ([essays-claude](../essays-claude/README.md)) в один движок — одна память, питающая один сценарный движок, который учится на собственных размеченных последствиях, — он становится частью «проекта, в котором реализуется само ядро». На бумаге эта сборка — центр корпуса; в портфеле это единственное, что ещё не построено.
