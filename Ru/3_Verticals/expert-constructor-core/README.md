# Expert Constructor Core — конструктор обоснованных доменных экспертных AI-чатов

> **Вверх:** [Вертикали — портфель](../README.md) · **English counterpart:** [`Eng/3_Verticals/expert-constructor-core/README.md`](../../../Eng/3_Verticals/expert-constructor-core/README.md)

> **Часть портфеля Real AGI.** Это слой ядра-движка: конфигурируемая платформа, которая превращает корпус документов в обоснованный экспертный чат, причём домен подключается конфигурацией, а не пересобирается каждый раз, — инфраструктура, из которой собираются вертикали. **Стадия:** рабочий прототип (запускаемый срез ответа; самообучение спроектировано, но ещё не в коде).

Expert Constructor Core — переиспользуемый конструктор экспертных чатов. Владелец знаний загружает документы; система готовит из них продвинутые пакеты знаний, извлекает релевантный контекст под каждый вопрос и составляет ответ, обоснованный этими источниками, через универсальный чат-интерфейс. Работа, которую он делает, узкая и честная: взять корпус и вопрос, вернуть ответ, заякоренный в корпусе. Он построен как пять модулей за явными контрактами — Chat UI, Knowledge Base Setup, Retrieval Context Builder, Answer Core Runtime и Synthetic Traffic Console, — так что любой из них можно заменить, не трогая остальные.

## Как это реализует концепцию

Он реализует слои *сборки эксперта* и *извлечения/обоснования* ядра, намеренно оставляя домен пустым. «Вертикаль» здесь задумана как **универсальное ядро + сменный пакет знаний + профиль**, что напрямую ложится на формулу корпуса: ядро + домен + рынок. На лестнице он достигает настоящего **пайплайна** (документы → пакет знаний → пакет извлечения → обоснованный ответ) с *первой половиной* агентного цикла обратной связи на месте: каждый ответ обязан испускать событие обратной связи, и существует eval-бенчмарк. Он не дотягивает до полноценного агента — замыкание цикла (коррекция поведения по накопленному опыту) спроектировано, но ещё не реализовано. Принципиально: это *эксперт, который отвечает*, а не *ментор, который ведёт*: менторские потоки явно вынесены за границу продукта.

## Что есть сегодня

Запускаемый Answer Chat UI, подключённый к Answer Core Runtime; детерминированная генерация пакета извлечения с тестами кэша/тематики/устаревания и eval-бенчмарком; Knowledge Base Setup, способный опубликовать локальный пакет и дескриптор доступа; версионированные промпты и архитектурный страж уровня проекта. Самообучение, живой межмодульный сервисный шов и рантайм Synthetic Traffic Console специфицированы, но не построены. Ни сегмент рынка, ни плательщик в материалах не зафиксированы.

## Куда это может вырасти

*Гипотеза:* если бы спроектированную подсистему обучения замкнуть (последствие → оценка → обновление правила), а отвечающий профиль сменить на ведущий — модель цели, веер сценариев, направляющая фигура, — этот движок становится субстратом, на котором инстанцируются вертикали Real AGI: одно ядро, много доменов, каждый входит как пакет знаний и профиль.
