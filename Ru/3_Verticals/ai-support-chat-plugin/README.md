# AI Support Chat Plugin — привязанный к базе знаний ассистент поддержки, который диагностирует, а не угадывает

> **Вверх:** [Вертикали — портфель](../README.md) · **English counterpart:** [`Eng/3_Verticals/ai-support-chat-plugin/README.md`](../../../Eng/3_Verticals/ai-support-chat-plugin/README.md)

> **Часть портфеля Real AGI.** Ранний, но полностью построенный агент (пайплайн + обратная связь + самообучение), который применяет паттерн «эксперт, который ведёт, а не просто отвечает» к одному узкому домену — клиентской поддержке — на одной платформе, WordPress. **Стадия:** работающий публичный релиз (v2.1.1, GPLv2), данных об использовании/выручке пока нет.

Плагин для WordPress, который добавляет чат-бот поддержки, отвечающий строго из выверенной базы знаний владельца сайта — без галлюцинаций, без ухода от темы. Вместо того чтобы выдать пользователю единственный заготовленный ответ, он классифицирует каждый вопрос, выдаёт проверенный ответ для простых случаев (T1) или проводит пользователя через многоходовой сценарий устранения неполадок для сложных (T2), где каждый ответ определяет следующий шаг. Когда он не может решить проблему, он эскалирует к человеку, а не импровизирует. Он поставляется с панелью аналитики, системой обратной связи из 4 параметров и админским ИИ-помощником. Это для владельцев малых и средних сайтов — e-commerce, edtech, SMB — которые сегодня платят за живых операторов или SaaS-чаты.

## Как это реализует концепцию

Это настоящий агент в смысле лестницы, ограниченный одной вертикалью. Пайплайн реален: сборщик контекста → LLM-роутер → ярусный ответ или ветвящийся сценарий → эскалация. Обратная связь замыкает цикл — ответы пользователя направляют следующий шаг, а оценки плюс история диалога стекаются в аналитику. Самообучение существует с человеком в цикле: модель анализирует накопленные диалоги и *предлагает* новые записи в базу знаний, которые администратор одобряет или отклоняет. Домен (кейсы поддержки) и рынок (WordPress SMB) явно присутствуют. Честно говоря, всё же он не дотягивает до полной вертикали Real AGI: сопровождение транзакционно — «реши эту одну проблему» — а не долгосрочное ведение к цели человека, и нет фигуры ментора. Он корректирует свою базу знаний, а не свою политику принятия решений.

## Что есть сегодня

Рабочий, прошедший аудит безопасности публичный плагин: роутинг T1/T2, 6-слойная контекстная память, эскалация, обратная связь, аналитика с ИИ-анализом, цикл самообучения через предложения, хранилище SQLite/Supabase и мультидоменное поле (поддержка/админ). Более полная премиум-версия живёт в отдельном приватном репозитории.

## Куда это может вырасти

*Спекулятивно.* Движок — память, сценарии, обратная связь, самообучение, абстракция хранилища — не зависит от домена; вынутый из оболочки WordPress, он мог бы стать ранним кандидатом-компонентом для общего ядра, где база знаний поддержки заменена на базу знаний любой вертикали.
